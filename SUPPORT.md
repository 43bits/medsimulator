# Support

For operational support, collect:

- facility and branch,
- user role,
- affected module,
- request time,
- request ID if available,
- screenshot or error message with patient data removed,
- whether the issue affects patient care, billing, payments, or reporting.

## Production Incident Priority

- P0: system unavailable, login unavailable, database unavailable, payments unsafe, or patient care blocked.
- P1: one critical module unavailable, M-Pesa delayed, reports blocked, or facility access incorrect.
- P2: broken screen, slow workflow, PDF issue, or non-critical data correction.
- P3: cosmetic, copy, documentation, or enhancement.

Security issues should follow [SECURITY.md](SECURITY.md).
